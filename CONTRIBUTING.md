# Contributing to Wombat Actions

First off, thank you for considering contributing to Wombat Actions! ✓ It's people like you that make these tools better for everyone.

## Getting Started

1.  Fork the repository (if you are an external collaborator).
2.  Create a new branch for your feature or bug fix.
3.  Make your changes, ensuring you follow the existing style and conventions.
4.  Test your action locally if possible, or create a test workflow.
5.  Submit a pull request.

## Adding a New Action

When adding a new action, please ensure you include:

- A self-contained directory for the action (e.g., `my-new-action/`).
- An `action.yml` file defining the action's inputs, outputs, and runs configuration.
- A `README.md` within the action's directory detailing its usage.

## Code of Conduct

Please be respectful and constructive in your interactions with the team and other collaborators.
